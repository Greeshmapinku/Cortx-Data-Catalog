================================================================================
CORTX DATA CATALOG - SETUP ON NEW SYSTEM (FRESH INSTALL)
================================================================================

This guide explains how to set up and run the Cortx Data Catalog on a 
completely new computer (Windows, Mac, or Linux).

================================================================================
STEP 1: INSTALL PYTHON
================================================================================

WINDOWS:
--------
1. Go to https://www.python.org/downloads/
2. Download Python 3.10 or higher (3.11 recommended)
3. Run the installer
4. IMPORTANT: Check "Add Python to PATH" checkbox
5. Click "Install Now"
6. Verify installation:
   Open Command Prompt and type:
   
   python --version
   
   You should see: Python 3.11.x or similar

MAC:
----
1. Open Terminal
2. Install Homebrew if not installed:
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   
3. Install Python:
   brew install python@3.11
   
4. Verify:
   python3 --version

LINUX (Ubuntu/Debian):
----------------------
1. Open Terminal
2. Update packages:
   sudo apt update
   
3. Install Python:
   sudo apt install python3.11 python3.11-venv python3-pip
   
4. Verify:
   python3 --version

================================================================================
STEP 2: DOWNLOAD THE PROJECT
================================================================================

OPTION A: Git Clone (if you have git installed)
-----------------------------------------------
1. Open Terminal/Command Prompt
2. Navigate to where you want the project:
   
   cd Documents
   
3. Clone the repository:
   
   git clone https://github.com/Greeshmapinku/Cortx-Data-Catalog.git
   
4. Enter the project folder:
   
   cd Cortx-Data-Catalog

OPTION B: Download ZIP (if no git)
----------------------------------
1. Go to https://github.com/Greeshmapinku/Cortx-Data-Catalog
2. Click green "Code" button
3. Click "Download ZIP"
4. Extract the ZIP file
5. Open Terminal/Command Prompt
6. Navigate to extracted folder:
   
   cd Downloads\Cortx-Data-Catalog-main
   (Windows)
   
   cd Downloads/Cortx-Data-Catalog-main
   (Mac/Linux)

================================================================================
STEP 3: CREATE VIRTUAL ENVIRONMENT
================================================================================

This creates an isolated Python environment for the project.

WINDOWS:
--------
python -m venv venv

MAC/LINUX:
----------
python3 -m venv venv

================================================================================
STEP 4: ACTIVATE VIRTUAL ENVIRONMENT
================================================================================

WINDOWS (PowerShell):
---------------------
.\venv\Scripts\Activate.ps1

If you get an error about execution policy, run:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Then try again:
.\venv\Scripts\Activate.ps1

WINDOWS (Command Prompt):
-------------------------
venv\Scripts\activate.bat

MAC/LINUX:
----------
source venv/bin/activate

You should see (venv) at the start of your command prompt.

================================================================================
STEP 5: INSTALL DEPENDENCIES
================================================================================

Make sure you're in the project folder and (venv) is active.

FOR LOCAL DEVELOPMENT (Full features):
--------------------------------------
pip install -r requirements.txt

This installs:
- flask, flask-cors (web server)
- pandas, pyarrow (data processing)
- pydantic (data validation)
- sentence-transformers (semantic search)
- groq (LLM annotation)
- And more...

NOTE: This will download ~500MB of packages including PyTorch.
It may take 5-10 minutes on slower connections.

FOR LIGHTWEIGHT/RRENDER MODE:
-----------------------------
pip install -r requirements-render.txt

This installs only essential packages (~50MB):
- flask, flask-cors
- pandas, pydantic
- pyarrow

================================================================================
STEP 6: VERIFY INSTALLATION
================================================================================

Check if key packages are installed:

pip list | findstr /i "flask pandas pydantic"
(Windows)

pip list | grep -i "flask\|pandas\|pydantic"
(Mac/Linux)

You should see flask, pandas, and pydantic in the list.

================================================================================
STEP 7: GENERATE CATALOG (First Time)
================================================================================

The app needs a catalog.json file to display data.

OPTION A: Use Pre-generated Demo Data
-------------------------------------
The repository includes catalog.json with demo data.
No action needed - skip to Step 8.

OPTION B: Generate Fresh Catalog with LLM
------------------------------------------
1. Get a Groq API key from https://console.groq.com
2. Set the API key:
   
   Windows:
   set GROQ_API_KEY=gsk_your_key_here
   
   Mac/Linux:
   export GROQ_API_KEY=gsk_your_key_here
   
3. Run the demo:
   
   cortx-catalog-gen --demo --source sqlite --uri demo_data\trading.db --annotate
   
4. This creates fresh catalog.json with LLM annotations

OPTION C: Generate from Northwind Dataset
-----------------------------------------
Use the provided dataset (no LLM needed):

python run_demo.py

This generates catalog.json from the Northwind CSV files.

================================================================================
STEP 8: RUN THE WEB DASHBOARD
================================================================================

1. Make sure (venv) is still active
2. Run the Flask app:
   
   python app.py
   
3. Wait for startup message:
   
   ==================================================
   STARTING CORTX DATA CATALOG
   ==================================================
   Loading catalog from catalog.json...
   ✓ Loaded 7 sources
   ==================================================
    * Running on http://127.0.0.1:5000
   
4. Open browser and go to:
   http://127.0.0.1:5000
   or
   http://localhost:5000

5. You should see the dark-themed dashboard with data sources

================================================================================
STEP 9: TEST THE API (Optional)
================================================================================

Open another terminal and test:

curl http://127.0.0.1:5000/api/health

Expected response:
{"status":"healthy","sources_count":7}

curl "http://127.0.0.1:5000/api/search?q=customer%20sales"

Expected: JSON with search results

================================================================================
STEP 10: STOP THE SERVER
================================================================================

1. Go back to the terminal running the server
2. Press: CTRL + C
3. Type: deactivate
4. Press Enter

================================================================================
COMMON ISSUES ON NEW SYSTEMS
================================================================================

ISSUE 1: "python" command not found
-----------------------------------
Solution: Use "python3" instead, or add Python to PATH during installation

ISSUE 2: "pip" command not found
--------------------------------
Solution: 
Windows: python -m pip install ...
Mac/Linux: python3 -m pip install ...

ISSUE 3: Permission denied when installing packages
---------------------------------------------------
Solution: Don't use sudo. The virtual environment handles permissions.
If issue persists: pip install --user <package>

ISSUE 4: "Microsoft Visual C++ required" (Windows)
--------------------------------------------------
Solution: Download and install:
https://aka.ms/vs/17/release/vc_redist.x64.exe

ISSUE 5: "No module named 'cortx_catalog'"
------------------------------------------
Solution: Install the package in editable mode:
pip install -e .

ISSUE 6: PyTorch installation fails
-----------------------------------
Solution: Install PyTorch separately:
pip install torch --index-url https://download.pytorch.org/whl/cpu

Then install remaining requirements:
pip install -r requirements.txt

ISSUE 7: Port 5000 already in use
---------------------------------
Solution: Edit app.py, change port 5000 to 5001:

port = int(os.getenv("PORT", 5001))

Then access: http://127.0.0.1:5001

================================================================================
QUICK START (FOR REPEATED USE)
================================================================================

Once set up, each time you want to run:

1. Open terminal
2. cd <project-folder>
3. Activate venv:
   Windows: .\venv\Scripts\Activate.ps1
   Mac/Linux: source venv/bin/activate
4. python app.py
5. Open browser: http://127.0.0.1:5000
6. CTRL+C to stop
7. deactivate

================================================================================
SYSTEM REQUIREMENTS
================================================================================

Minimum:
- Windows 10/11, macOS 10.15+, or Ubuntu 20.04+
- 4 GB RAM (8 GB recommended)
- 2 GB free disk space
- Python 3.10 or higher
- Internet connection (for initial setup)

Recommended:
- 8 GB RAM
- SSD storage
- Python 3.11
